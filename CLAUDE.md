## 開発ワークフロー（重要）

### プラン → 実装 → レビュー の3ステップを必ず守ること

```
[上位モデル] プランを作成・設計判断
    ↓ 明確なゴールと完了条件を渡す
[サブエージェント] 実装（コーディング・テスト作成）
    ↓ 完了報告を返す
[上位モデル] レビュー・承認 or 修正指示
    ↓
[CLI] lint / typecheck / test（トークン消費なし）
```

### 上位モデル（あなた）の責務
1. **まずプランを立てる** — いきなりコードを書かない
2. 影響範囲を特定し、実装方針を決める
3. サブエージェントに明確なゴールと完了条件を渡す
4. サブエージェントからの報告を必ずレビューする
5. 設計意図との整合性・エッジケースの考慮漏れをチェックする

### サブエージェントへの委譲対象
- ファイルの新規作成・編集
- テストの作成・実行
- 依存関係のインストール
- リファクタリングの実行

### サブエージェントに返させる情報
- 変更したファイル一覧
- 主要な実装判断とその理由
- 未解決の懸念点

### 実行チェックリスト（毎タスク確認）
1. [ ] プランを先に立てた？（変更するファイル・方針・完了条件）
2. [ ] 実装はサブエージェントに委譲した？
3. [ ] サブエージェントの報告をレビューした？
4. [ ] `npm run lint` を実行した？
5. [ ] `npm run build` を実行した？

## コスト最適化

### LLMを使わない処理
以下は必ずCLIツールで直接実行（トークン消費ゼロ）：
- lint
- format
- type check
- test実行

具体的なコマンドは「開発コマンド」セクションを参照。

### コンテキスト管理
- 長いログ出力はサブエージェントに閉じ込める
- 適宜 /compact を使う
- タスク完了後は /clear を検討

---

## コーディングルール

### 全般
- 型アノテーションを必ず付ける
- 簡潔で読みやすいコードを優先する
- 関数・クラスにはdocstringを書く
- マジックナンバーは定数化する
